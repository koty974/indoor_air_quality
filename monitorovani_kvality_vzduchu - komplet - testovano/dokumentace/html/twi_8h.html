◆ DDR

#define DDR ( _x)
Value:
(*(&_x - 1))

AVR trick macros: convert a PORT register name to its corresponding Data Direction (DDR) and PIN registers.

Example: DDR(PORTC) → DDRC PIN(PORTC) → PINC

These work because of the way the I/O registers are organized in memory: DDRx is always located one address below PORTx. PINx is always located two addresses below PORTx.

◆ TWI_BIT_RATE_REG

#define TWI_BIT_RATE_REG   ((F_CPU/F_SCL - 16) / 2)

Formula for TWI bit rate: SCL_freq = F_CPU / (16 + 2 * TWBR * Prescaler) TWI_BIT_RATE_REG computes the correct TWBR value given F_CPU and F_SCL.

◆ twi_readfrom_mem_into()

void twi_readfrom_mem_into ( uint8_t addr,
uint8_t memaddr,
volatile uint8_t * buf,
uint8_t nbytes )

Read multiple bytes from a slave device starting at a memory/register address.

Parameters
addrSlave address.
memaddrStarting memory/register address.
bufPointer to buffer for received data.
nbytesNumber of bytes to read.

Performs the following sequence:

  1. Send START + SLA+W
  2. Send memory address
  3. Send repeated START + SLA+R
  4. Read N bytes (ACK for all but last, NACK for last)
  5. Send STOP

Send one byte (address or data) on the TWI bus.

Parameters
dataThe byte to transmit.
Returns
0 if ACK received, 1 if NACK.
Note
Returns 0 (ACK) if TWI status register indicates:
  • 0x18: SLA+W transmitted, ACK received
  • 0x28: Data byte transmitted, ACK received
  • 0x40: SLA+R transmitted, ACK received
